## The Honor Code and CPSC 220

All graded work is covered by the conditions of the UMW Honor Code. If you have any questions as to what is allowed, ask me.  You must adhere to all tenets of the CPSC Honor Code Policy, which is given at the following link.

[CPSC Honor Code Policy](http://cs.umw.edu/mediawiki/index.php/CPSC_Department_Honor_Code_Policy)

You shall provide the UMW Honor Code Pledge with all assignments.  If the assignment is an electronic submission, you shall type the pledge along with your name.  If the assignment is a printed submission, you shall handwrite the pledge along with your signature.

* Exams: You shall NOT discuss exam questions or share answers in any way with others.  If you require clarification during an exam, ask me a question.

* Labs: You may freely collaborate with your team members on all laboratory assignments.  The collaboration can include explaining the problem, explaining your code, demonstrating your code, explaining a question, and explaining your answer to a question.  The only constraint on lab team collaboration is that it must result in learning.  The collaboration shall not be one person copying the code of another.  After a discussion where a team member has helped another understand a programming problem or a question, a student must (1) write the code independently (no copying/sharing files) and (2) write the answers to lab questions independently.  You shall NOT search the Internet for solutions.

* Programming Projects: Programming projects contain individual assignments components and team components.  You may collaborate with your team for the team components.  You must solve the individual components yourself, using your brain and the information presented in class.  You shall NOT consult with others and you shall NOT search the Internet for solutions.  You may consult with others on syntax errors; however, if you need assistance on semantics, talk with me. 

* Java Documentation: You can use the Java documentation on the Oracle website.  The following link is a good starting point:  http://docs.oracle.com/javase/8/ 

Please review the list of computer science specific honor code violations below. This is not intended to be an exhaustive list of possible violations.

1. Accessing another student's files (in printed or electronics format) that are related to programming projects.
2. Allowing another student to access your files (in printed or electronic format) pertaining to programming projects.
3. Falsifying information to make your instructor think that your program works differently than it actually does.
4. Receiving / giving assistance from / to others that involves the logical design or implementation of a current programming project.
4. Using code written by others, which includes code found on the Internet.
