CPSC 405 Lab Submissions

Submitting Labs

Labs are designed to improve your computer science skills and knowledge. The labs are a learn by doing component of our course and involve significant programming.

We have two types of labs. The majority of our labs are Xv6-based Labs. These labs are completed by adding code to Xv6, building, running, and testing your Xv6 code. We have a few Linux-based Labs. These labs are completed by creating standalone C programs that run on Linux.

Lab Questions

Most of our labs have questions that you must answer. Questions are in boxes (like this) with a light orange background.

  • You must write each question and its answer in your notebook. You can abbreviate some of the question, but you must write enough so that the question is understandable.
  • You must write you lab questions and answers starting on a fresh page of your notebook.
  • For lab submissions, you must take photo(s) of your questions/answers and submit the photo(s) on Canvas.
  • Additionally, in the directory of your xv6-labs, create the file: answers-LAB.txt, where the string LAB corresponds to the lab you are submitting. For example, answer-util.txt is the file for the util lab. You do not need to put answers in this file, but the grading script checks to see if this file exists. If it exists, I assume you have answered the questions in your notebook, and you have submitted photos to Canvas.

Submitting Labs

Each team member shall submit lab information on Canvas.

When you have completed your lab, on Canvas you must submit

  • a Zip file.
  • a Text file.
  • a Photo file.
Items are marked with Xv6, Linux, or Both to indicate whether the item applies to Xv6 Labs, Linux Labs, or Both.

Zip File

The Zip file contains the source code, Makefile, etc., of your lab.

  • Xv6: A zipped tar file of you lab code. The name tar is short for Tape ARchive, and in the past magnetic tapes were used to archive files for safe keeping. The Linux utility tar creates tar files.
    • To create the zipped tar file, run make tarball in your lab directory, which creates (for example) the file lab-util-handin.tar.gz, where util is the name of the lab. The name of the .tar.gz file depends upon the lab. Submit this file on Canvas.
  • Linux A Zip file of your directory. Submit the resulting .zip file to Canvas.

The following image shows Ritchie and Thompson working on Linux using a PDP computer. The orange lines encircle tapes that would have been used to backup code in an archive.

Text File

You submit one text file. The text file name shall be for example, lab-util-handin.txt, where util is the name of the lab. The text file shall be neatly organized items in the following order.

  • Both: Name: Your name.
  • Both: Hours Spent: The number of hours you spent on the lab.
  • Both: Assessment: An assessment of your effort on the lab.
  • Both: Grade: A grade you deserve on the lab.
  • Both: Teammates Assessment: An assessment of you teammates' efforts on the lab.
  • Both: Teammates Grades: A grade your teammates deserve on the lab.
  • Xv6: git status: A copy/paste of the output of git status. All of your changes should be committed.
  • Xv6: make grade: A copy/paste of the output from make grade. If you have uncommitted changes, explain them.
